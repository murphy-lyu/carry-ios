# 真机验收 Checklist（2026-06-02 QA 全量修复后）

> 用真机或模拟器逐项勾选。**带 📱** 的必须真机（模拟器不复现）；其余模拟器可验。
> 失败时记下"哪一步、看到什么、预期什么"，告诉我即可。

---

## A. 数据同步链（PR #32）

- [ ] 创建一个有日期的行程 → 开「日历同步」 → 在系统日历看到事件
- [ ] 编辑该行程的名字 → 系统日历事件标题**同步更新**
- [ ] 编辑该行程的日期 → 系统日历事件日期**同步更新**
- [ ] 把该行程改为「未来某天」（清空日期）→ 系统日历事件**消失**
- [ ] 删除该行程 → 系统日历事件**消失**
- [ ] 复制该行程 → 系统日历多一条事件 + 复制的行程到出发日**收到打包提醒**

## B. 备份还原（PR #33 + #34）

- [ ] 设置 → 数据 → 导出备份 JSON → 文件能分享/保存
- [ ] 删几个行程 + 改设置 → 导入刚才的备份 → **数据完整还原**（含多目的地、通知偏好）
- [ ] 故意改坏备份文件（用文本编辑器把 `"version":1` 改成 `"version":99`）→ 导入 → **应提示"请先更新 App"**（而非"文件损坏"）
- [ ] 还原后检查：旧通知**全消失**，灵动岛**不挂老行程**（如果之前有）
- [ ] 用应用数据目录的 `carry_backup.pre-restore-{epoch}.json` 验证：执行还原后该文件存在（手动从 App 沙盒检查；正常用户不需要看）

## C. LiveActivity（PR #35）📱

- [ ] 出发前 3 天内打开行程 → 锁屏 / 灵动岛**出现 Live Activity**
- [ ] 勾选物品 → 灵动岛**进度同步更新**
- [ ] 快速切换 A → B → C 三个行程 → 灵动岛**始终对应当前打开的行程**，不重叠/不残留
- [ ] 📱 跨时区测试：手动改本机时区到目的地（设置 → 通用 → 日期与时间 → 关闭自动 + 改时区）→ Live Activity **不应在出发当天 0 点就消失**，应在出发次日才消失
- [ ] 删除一个有 LA 的行程 → 灵动岛**立即消失**
- [ ] 设置 → 重置 → 灵动岛**全部消失**

## D. 通知调度（PR #36）📱

- [ ] 设置默认提醒「出发当天 + 出发前 1 天」 → 创建一个**明天出发**的行程 → 系统设置看排了 2 条通知
- [ ] 编辑该行程把提醒时间改成 X 分钟后 → 等待 → **应收到一次通知**
- [ ] 在「出发前 1 天 9:00」的提醒**已过去**的情况下（比如今天下午编辑了"出发前 1 天"提醒，明天就出发，9:00 已过）→ **60 秒后应收到一次降级通知**
- [ ] 关掉行程的总开关 → 改某档时间 → **不应**重排出新通知（系统设置看不到）
- [ ] 把行程改为「未来某天」→ **所有通知应被取消**
- [ ] 📱 跨时区：在北京设置「09:00 提醒」→ 改本机时区到纽约 → 提醒应仍按**北京时区 09:00** 触发（约纽约 21:00），而非纽约 9:00

## E. 错误处理日志（PR #37）

- [ ] 拒绝通知权限 → 打开行程 → 不应崩；日志（DEBUG 控制台）应见 `reminder_schedule_failed`
- [ ] 拒绝日历权限 → 打开日历同步 → 友好提示，不崩；日志应见 `calendar_save_failed`
- [ ] 切飞行模式 → 进有目的地的行程 → 天气卡空白不崩；日志应见 `api_error`

## F. 深链冷启动（PR #38）📱

- [ ] 完全杀掉 App → 长按图标 → 选「最近行程」→ **能正确打开那个行程**（不停在首页）
- [ ] 完全杀掉 App → 添加 Carry Widget → 点 Widget 上某个行程 → **能正确打开那个行程**
- [ ] 完全杀掉 App → 收到打包提醒通知 → 点通知 → **能正确打开对应行程**

## G. Minor 修复（PR #40）

- [ ] 有未来行程 + 过去行程时,长按图标 → 「最近行程」→ 打开的应是**最近未来**的(非昨天刚结束的)
- [ ] 行程已勾过的清单 → 改某个 section 标题 → 重选场景(regenerate)→ 该 section 的自定义物品应**进入「其他」section 而非消失**
