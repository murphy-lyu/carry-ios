# 单个行程花费：综合费用 · 分布 · 详细清单

Status: Draft（待确认，2026-06-21）

## 目标
在**单个行程**内查看：① 综合总花费；② 费用分布（按类别 / 按天 /（多币种时）按币种）；③ 逐笔详细清单。
诚实优先于炫目：费用是**可选录入**，本页呈现的是「**用户已记录**的花费」，不假装是完整预算。

## 与现有架构的关系（复用，不重载）
- **费用真相**：`CostBearing`（`costAmount` + `costCurrencyCode` 原币种=真相、永不丢；`costHomeAmount` 本位币快照）。spec: `itinerary-cost-tracking.md`。
- **可记费用的实体**：`ItineraryStop`（地点，带 `StopCategory`）、`TransportSegment`（交通段，`TransportMode`）、`LodgingStay`（住宿）。
- **折算口径复用 `CostResolver.homeValue`**：优先快照 → 无快照退实时折算 → 再无汇率则「未计入并诚实标注」（同 `TripSpendStats`）。
- **新增一个「单行程」聚合器**（如 `TripSpendDetail`），**不复用** `TripSpendStats.compute`（那是 Trip Book 跨行程、只统计已发生行程、且只分粗三类）。本页要：任意单行程（含未发生）、更细类别、按天/币种维度、逐笔清单。纯函数、可单测。

## 类别口径（展示用 7 类）
由「实体类型 + `StopCategory`」归并为固定 7 类（颜色用 app 既有类别色）：

| 展示类别 | 来源 |
|---|---|
| 交通 | `TransportSegment`（航班/火车/巴士/渡轮/租车，**合并为一类**——再细分进逐笔清单看，不在分布里堆 11 类） |
| 住宿 | `LodgingStay` |
| 餐饮 | `ItineraryStop` 且 `category == .food` |
| 景点门票 | `.sightseeing` |
| 活动 | `.activity` |
| 购物 | `.shopping` |
| 其他 | 其余 stop 类别（含 `.other` 及罕见的非地点类别落到 stop 上的情况） |

0 元/无记录的类别**不显**（不堆空行）。

## 入口与定位
- **行程列表底部一张「花费」汇总卡**（常驻）：`本次行程 ¥X · 已记录 N 笔 >`，点进**专属花费页**（push 或 sheet）。
- **不做成第三个 Tab**：Itinerary/Packing 切换器是「干活面」，花费是「回看面」；汇总卡「一眼可见 + 一点深入」更克制，且与 Trip Book（跨行程回顾）形成「单程详看 / 全局回顾」分工。
- 单行程**零记录**时：汇总卡退化为轻量入口（`记录花费 >` 或不显总额），不显误导性的「¥0」。

## 专属花费页结构（一条滚动，一个控件统领）
1. **Hero 总额**：大号本位币总额（圆体数字）。meta 行 `N 笔 · M 天 · 日均 ¥Z`。文案用「**已记录花费**」。
2. **比例细带**：紧贴总额下方一条横向堆叠比例带（类别色），秒懂格式塔（大头是谁）。**不用饼图**（读不出精确值、显花哨）。
3. **维度切换（segmented）**：`类别 / 时间`（**多币种时**多一个 `币种`）。**同一控件同时重排「分布图」和「下方清单」**——切到「时间」上面变按天柱状、下面清单按天分组。一个控件、整页跟着变。
4. **分布（按所选维度）**——钱包/Apple Card 式「类别行」：
   - 每行 `色点 + 名称 + 占比细条 + 金额 + 百分比`，按金额降序。条形+精确数字，好扫又能读准。
   - 「时间」= 每天一根柱（哪天最烧钱）；「币种」= 各币种各花了多少**原币**。
   - 点某条 → 下方清单滚动/过滤到该组（分布↔清单联动）。
5. **详细清单（按所选维度分组）**：
   - 每行 `类别图标 + 名称（地点/酒店/航班…） + 原币金额`；若原币 ≠ 本位币，副行显本位币折算。
   - **点任一条 → 跳到该项详情/编辑**（闭环：发现某笔不对当场改）。

## 多币种：诚实第一
- 总额 = 各笔**本位币快照之和**（`costHomeAmount`，缺则实时折算）；底部小字「总额按**记录当时汇率**折算为 ¥」。合计含外币折算时总额前加「≈」。
- 清单永远显**原币真相**，折算仅作副行。
- `币种` 维度**仅 ≥2 种币种时才出现**；单币种不堆此 chip。
- 有费用因**无汇率未计入**合计 → 脚注诚实标注「另有 K 笔因缺汇率未计入」（同 `TripSpendStats.hasUnconverted`），绝不静默漏算。

## 数据 / 工程
- **无 schema 改动**：纯聚合派生（读 `CostBearing` 三字段 + `StopCategory` / `TransportMode`）。
- 聚合在 `TripSpendDetail.compute(trip:homeCode:convert:)`（纯函数、单测），UI 只渲染。
- 折算注入 `convert`（解耦汇率源、便于单测），同 `TripSpendStats` 范式。
- 性能：单行程数据量小，前台同步聚合即可；行程变动时重算（@Published/计算属性）。

## 边界 / 空态
- 零记录：花费页空态「还没记录花费，在每个地点/交通/住宿里填金额即可」。
- 单天行程：「时间」维度仍可用（就一天），或该 chip 退化（仅一天时分布=总额，可只显类别）。
- 无日期行程（dateless）：「时间」维度按「第 N 天」分组（无真实日期）。
- 同一实体多次出现（住宿跨天）：费用**只计一次**（按实体，不按天展开计费）。

## 验收（实现后）
- 单行程有多类多币种费用时：总额正确、比例带与类别行占比一致、切「时间/币种」整页正确重排。
- 点类别 → 清单联动；点某笔 → 跳到对应编辑页。
- 多币种：原币真相不丢、折算有「≈」与脚注；缺汇率的笔有诚实标注。
- 零记录 / 单天 / 无日期 / 住宿跨天：均无异常、不重复计费。
