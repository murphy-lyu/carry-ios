# Itinerary Export Document（签证行程单 — 双语 PDF 导出）

> **Status: Draft → implementing。** 设计已在需求讨论中拍板（见下「已确认决策」），本 spec 记录落地。

## 动机

把用户排好的行程（地点 + 交通 + 住宿）导出成一份**正式、可打印、可交签证**的逐日行程单。
这是「行程规划」这条线的兑现：办签证时使馆要的「day-by-day travel plan / cover letter」。

**诚实定位（对外话术务必准确）**：这是签证材料里的**一份**——「行程说明」，**不是**机票/酒店预订凭证、
更不是「一键搞定签证」。Carry 只如实呈现用户已录入的计划，不编造预订号、不声称官方效力。

## 已确认决策（来自需求讨论）

1. **申请人头部**：姓名 + 出行目的 + 起止日期；**坚决不碰护照号**（敏感个人信息，触发 PIPL 义务）。
   导出时填、本地可选存、不上云、不强制。
2. **格式只做 PDF**：使馆只认可打印 PDF。**不做 Excel**（使馆不收）、**不做 JPEG**（海报已覆盖「好看可分享图」）。
3. **双语**：导出时选 **EN 或 ZH** 一种（EN 交签证、ZH 自留）。文档框架文案 + 日期格式按所选语言渲染，
   **用户数据（地点名/航司/酒店名）原样带出、不翻译**。中英对照（side-by-side）列为后续。
4. **概览地图默认带上**、可关（导出选项里的开关），复用 `TripShare.renderRouteMap`。

## 数据来源（均已就绪）

- 行程：`TripBundle`（name/destinationCity/departureDate/isDateless/spanDays）
- 逐日：`ItineraryDay.timeline`（停靠点 `ItineraryStop` + 交通段 `TransportSegment`，按 sortOrder）
- 住宿：`TripBundle.safeLodgingStays`（`LodgingStay`，check-in/nights）
- 概览图：`TripShare.renderRouteMap(for:)`（async，`MKMapSnapshotter` + 图钉 + 动线）

## 实现

### 1. 双语文档标签（`ItineraryDocumentText.swift`）

文档内**固定文案**（标题、字段名、Day、入住/退房、页脚等）须按**所选语言**渲染，
与设备 locale 无关 → 用代码内 EN/ZH 小字典（仅这两种语言，导出二选一），**不进 xcstrings**
（xcstrings 走设备 locale，无法按选定语言取值）。日期/时间用 `Locale(identifier:)` 对应所选语言格式化。

### 2. PDF 渲染（`ItineraryPDFRenderer.swift`）

- `UIGraphicsPDFRenderer`，A4（595×842pt），页边距 48pt。
- 维护 y 游标逐块绘制 `NSAttributedString`；超页则 `beginPage()` 换页（一天的标题尽量不与首行分页）。
- 结构：
  - **头部**：标题「Travel Itinerary / 旅行行程单」+ 行程名 + 目的地 + 起止日期；可选申请人（姓名 / 目的）。
  - **概览地图**（可选）：等宽缩放绘入。
  - **逐日**：「Day N · 周几 月/日」→ 该天 timeline：交通段（mode + 班次 + 起讫站/时间，跨天标注）、
    停靠点（时间 + 名称 + 地址）、住宿（入住/退房事件）。
  - **住宿汇总**（trip 级，列每段：名称 / 入住–退房 / 晚数 / 地址）。
  - **页脚**：「Generated by Carry · carry app」+ 页码。
- 异步：若含地图先 `await TripShare.renderRouteMap` 再绘制；返回 `Data`（PDF）。

### 3. 导出选项 UI（`ExportItinerarySheet.swift`）

- Form：语言（EN / ZH 分段）、申请人姓名（选填）、出行目的（选填）、包含路线地图（开关，默认开）。
- 「导出 PDF」→ 渲染 → `UIActivityViewController`（或 `ShareLink`）分享/存储。
- 文件名：`行程名_Itinerary_yyyyMMdd.pdf`。
- 申请人信息可选「记住」到 `UserDefaults`（本地、不上云）。

### 4. 入口

`PackingListView` 行程「…」菜单，`detailTab == .itinerary` 段，与「分享行程 / 发送给朋友」并列：
「导出行程单 / Export itinerary document」。`hasShareableItinerary` 为真才可用。

### 5. 埋点

`CarryLogger`：`itineraryExported`（context: lang + hasMap）。错误 `itineraryExportFailed` 入 errorEvents。

## 本地化

- **导出选项 UI**（chrome）：结构化 key × 9 语言，走 xcstrings（设备 locale）。
- **文档内固定文案**：EN/ZH 代码字典（见上），不进 xcstrings。

## 合规

- 不含护照号；申请人信息本地、可选、不上云。
- 不声称官方效力 / 不生成预订凭证。

## 后续（非本轮）

- 中英对照（side-by-side）版；Excel 数据导出（如有呼声）；申请人信息更多字段。
