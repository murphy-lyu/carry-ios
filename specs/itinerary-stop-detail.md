# Itinerary Stop Detail（停靠点：只读详情态 + 导航收进模块）

> **Status: Draft — 待确认，未实现。** 本 spec 定义交互改动，确认后再编码。
>
> **与并行工作的关系（重要）：** 本改动落在 `StopEditView`、停靠点行结构、`itinerary-stop-navigation.md` 之上，而这片正由 `itinerary-transport-lodging.md`（交通段/住宿/「+」菜单）的并行开发占用。**实现须与该并行工作协调时机**（或等其落定后），避免共享工作区的合并冲突。

## 动机

当前点击行程里的某个停靠点（整行点击）→ **直接进入 `StopEditView`（可编辑态）**。两个问题：

1. **高频意图被低频意图绑架**：这屏点进来**绝大多数是为了「看」**（几点、在哪、备注），「编辑」是少数。默认落在可编辑态，把高频「查看」和低频「编辑」混成一个态，违反「信息密度服务当前任务」。
2. **易误操作**：直接可编辑，用户一进来就可能误改时间/位置这类敏感字段。

并行的「行内导航按钮（↗）」（见 `itinerary-stop-navigation.md`）放在**每行最外层**，而行内信息已很密（名称 + 地址 + 备注 + 时间）。导航其实是「看了地点之后才偶尔想用」的二级动作，放外层既加重信息过载、又和行尾时间抢位。

**参考 Tripsy**：点地点 → 只读详情卡（地址/电话/费用/备注/「旅行时间」模块），右上角才是编辑；导航/路程收在详情里的模块。本 spec 借其**交互结构**，用 Carry 克制审美决定**范围**（不抄电话/费用分摊/附件等 Carry 没有的 per-stop 字段）。

## 核心设计原则

1. **查看与编辑分态**：点停靠点 → **只读详情态**；编辑是详情里的显式入口，不是默认落点。
2. **导航降级为详情内模块**：从「每行外层操作」收进「详情内的到达/路程模块」，行尾因此腾空。
3. **克制**：只读详情只展示 Carry 真有的字段，不为对齐 Tripsy 而堆模块。
4. **不破坏现有编辑能力**：`StopEditView` 不动，只是改成「从详情里进」。
5. **顺平台**：只读详情用原生 sheet；导航复用现有 `MapNavigationService`（Apple/高德/Google/百度），不自造。

## 交互流

```
点击停靠点整行
   → 弹只读详情 sheet（StopDetailView）        ← 改动点：原来直接弹 StopEditView
        · 右上「编辑」→ 进 StopEditView（现有）
        · 详情内「导航」按钮 → MapNavigationService 弹应用选择
   左滑：保持现状（只删除）
```

## StopDetailView（新建 · 只读详情）

只读、克制，只放 Carry 真有的字段：

- **头部**：类别图标（与时间轴圆点同款）+ 地点名（大号圆体）+ 当天日期。
- **时间**：开始–结束（只读展示；未设时间则不显示此行，不留空块）。
- **地址**：完整地址（不截断）；可点 → 在系统地图打开 / 拷贝（二选一，建议「在地图打开」）。
- **备注**：完整展示（**不**像列表里截断两行）。空则不显示。
- **到达 / 路程模块**（导航的新家，见下）。
- **右上「编辑」** → 现有 `StopEditView`（sheet 或 push，跟随现有模态规范 `app-navigation-framework.md`）。

> 明确**不做**（Carry 无此 per-stop 数据，不为对齐 Tripsy 强加）：电话、总费用、附件/文件、多种交通方式时长矩阵（步行/驾车/公交/骑行四态）。如未来加，再单列。

## 导航 / 路程模块（从行尾迁入详情）

- 取代每行的 ↗ 按钮。位置在 `StopDetailView` 底部一个模块。
- **内容（Carry 版，克制）**：
  - 到**下一站**的距离（复用现有 leg 距离计算；末站无下一站则只给「导航到此地」）。
  - **「导航」按钮** → `MapNavigationService.availableApps()`：≥2 个应用弹选择菜单、仅 1 个直接调起（沿用 `itinerary-stop-navigation.md` 现有逻辑，仅换承载位置）。
- **无坐标**：按现状显示「无法定位」提示（`mappin.slash`），导航按钮不出现。

## 停靠点行（列表）简化

- **移除行尾 ↗ 导航按钮**（迁入详情模块）。
- 行尾腾空后，**开始–结束时间**贴到真正的行尾（当前因和 ↗ 抢位，时间被挤在其左侧）。名称/地址/备注布局不变（名称行右对齐时间，见已实现的「时间移到名称行」改动）。
- 无坐标标记（`mappin.slash`）何去何从：可保留在行内作为「此点无法导航/定位」的轻提示，或一并收进详情——**待定**（倾向保留行内轻提示，因为它是「数据完整性」信号、看列表时就该知道）。

## 范围 / 不做

- 不加 per-stop 电话/费用/附件。
- 不做多交通方式时长矩阵。
- `StopEditView` 内部不重构，仅改「从详情进入」。
- 导航能力本身不变（仍是 `MapNavigationService` 那套），只换触发位置。
